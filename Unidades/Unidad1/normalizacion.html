📘 Semana 4

Trabajo de Normalización realizado a una boleta electrónica

En esta actividad se aplicó el proceso de normalización de bases de datos tomando como referencia una boleta electrónica real, identificando entidades, atributos y relaciones necesarias para estructurar la información.

🧾 Boleta Electrónica

Se analizaron los datos visibles del comprobante para reconocer campos importantes como cliente, productos, fecha y atención.
Luego se trasladó la estructura inicial al software ER/Studio para comenzar el diseño del modelo relacional.

📌 Sin Normalización: Boleta

Primero se agruparon todos los datos tal como aparecen en la boleta, detectando redundancias y atributos mezclados dentro de una sola tabla.

👤 Forma Normalizada 1: Cliente

Se separó la información correspondiente al cliente para evitar repetir datos personales en cada compra registrada.

📦 Forma Normalizada 2: Producto

Los productos fueron organizados en una tabla independiente con sus características y detalles comerciales.

🧑‍💼 Forma Normalizada 3: Atención

Finalmente se estructuró la información relacionada con la venta, atención realizada y vínculo entre cliente y productos.
