Normalización de una Ficha de Inventario

Desarrollo técnico del proceso de normalización aplicado a una ficha de inventario informático institucional, identificando entidades, atributos, relaciones y estructuras óptimas en bases de datos.

Ficha Original de Inventario

Fuente de análisis:
Documento físico utilizado para registrar equipos tecnológicos institucionales.
Información encontrada:
Usuario, dependencia, hardware, periféricos, software y estado operativo.
Objetivo:
Transformar la ficha manual en una base de datos relacional profesional.
Resultado esperado:
Mejor control, mantenimiento y escalabilidad.

0FN - Tabla Sin Normalizar

Todos los datos se encuentran mezclados en una sola estructura.

Datos repetidos.
Campos múltiples en una sola fila.
Difícil mantenimiento.
No escalable.

Primera Forma Normal (1FN)

Cada campo contiene un solo valor y se separa usuario de equipo.

Segunda Forma Normal (2FN)

Se independizan tablas relacionadas con hardware y periféricos.

Tercera Forma Normal (3FN)

Modelo final profesional eliminando dependencias transitivas.
