Consultas Básicas en SQL Server

Desarrollo de consultas SQL aplicadas a la base de datos InventarioComputadores, utilizando operaciones básicas de búsqueda, filtrado, ordenamiento y relaciones entre tablas.

Contexto

En esta práctica se aplicaron diferentes consultas SQL utilizando la base de datos desarrollada previamente. El objetivo fue reforzar el uso de sentencias básicas en SQL Server para manipular y consultar información de un sistema de inventario tecnológico.

Casos de Consultas SQL 

-- CASO 1: Mostrar todas las dependencias registradas
SELECT * FROM DEPENDENCIA;

Esta consulta recupera el listado completo de todas las dependencias registradas dentro de la base de datos, mostrando información como su código y nombre correspondiente. 

-- CASO 2: Mostrar todos los usuarios registrados
SELECT * FROM USUARIO;

Esta consulta muestra todos los usuarios almacenados en la tabla USUARIO, permitiendo visualizar la información completa registrada de cada empleado dentro del sistema, así como su relación con la dependencia correspondiente. 

-- CASO 3: Mostrar todos los equipos tecnológicos
SELECT * FROM EQUIPO;

Esta consulta muestra todos los equipos tecnológicos registrados dentro del inventario. Permite visualizar información como tipo de equipo, marca, modelo, número de serie, estado y demás datos asociados al hardware almacenado en la base de datos. 

-- CASO 4: Mostrar solamente los nombres de las dependencias
SELECT nombre_dependencia FROM DEPENDENCIA;

Esta consulta obtiene únicamente los nombres de las dependencias registradas en la empresa. A diferencia de la consulta completa, aquí solo se muestra el campo nombre_dependencia, dejando de lado códigos o identificadores internos. 

-- CASO 5: Mostrar el nombre del usuario y su dependencia
SELECT U.nombre_usuario, D.nombre_dependencia 
FROM USUARIO U
JOIN DEPENDENCIA D ON U.id_dependencia = D.id_dependencia;

Esta consulta relaciona las tablas USUARIO y DEPENDENCIA mediante JOIN, mostrando el nombre de cada usuario junto con la dependencia a la que pertenece. El resultado facilita identificar rápidamente qué trabajador está asignado a cada área dentro de la organización. 

-- CASO 6: Mostrar todos los equipos tipo LAPTOP
SELECT * FROM EQUIPO WHERE tipo = 'Laptop';

Filtra el inventario para listar únicamente los dispositivos de tipo portátil, permitiendo identificar rápidamente cuántas laptops hay asignadas. 

-- CASO 7: Mostrar los equipos marca HP
SELECT * FROM EQUIPO WHERE marca = 'HP';

Localiza todos los activos tecnológicos fabricados por la marca HP, útil para auditorías de proveedores o gestión de garantías. 

-- CASO 8: Mostrar los equipos con estado BUENO
SELECT * FROM EQUIPO WHERE estado = 'Bueno';

Esta consulta permite visualizar la información solicitada desde la base de datos InventarioComputadores, mostrando los registros correspondientes según las condiciones establecidas en el script SQL.

El resultado obtenido confirma el correcto funcionamiento de las sentencias SQL aplicadas en SQL Server.

-- CASO 11: Mostrar los equipos ordenados por marca alfabéticamente
SELECT * FROM EQUIPO ORDER BY marca ASC;

Esta consulta muestra todos los equipos registrados en la tabla EQUIPO, ordenándolos alfabéticamente según el campo marca mediante la cláusula ORDER BY ASC.

El resultado permite visualizar el inventario organizado por marca, facilitando la búsqueda, clasificación y revisión de los equipos tecnológicos almacenados dentro de la base de datos. 

-- CASO 12: Mostrar los equipos ordenados por fecha de registro (más recientes primero)
SELECT * FROM EQUIPO ORDER BY fecha_registro DESC;

Esta consulta muestra todos los registros de la tabla EQUIPO ordenados según la fecha de registro mediante ORDER BY fecha_registro DESC.

El resultado presenta primero los equipos más recientemente registrados dentro del inventario, permitiendo identificar rápidamente las últimas incorporaciones realizadas en la base de datos.

-- CASO 19: Mostrar la cantidad de equipos que tiene asignada cada dependencia
SELECT id_dependencia, COUNT(*) AS Cantidad_Equipos 
FROM EQUIPO 
GROUP BY id_dependencia;

Esta consulta agrupa los registros de la tabla EQUIPO según la dependencia asignada mediante GROUP BY id_dependencia, contando cuántos equipos pertenecen a cada una.

El resultado permite conocer la cantidad total de equipos asignados por dependencia, facilitando el control del inventario y la distribución de recursos tecnológicos dentro de cada área de la organización.
