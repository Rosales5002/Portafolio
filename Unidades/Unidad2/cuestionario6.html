Cuestionario Teórico

Preguntas prácticas desarrolladas sobre conceptos, administración, seguridad y rendimiento en Microsoft SQL Server.

1. ¿Por qué es crítico definir claves primarias y foráneas?
Las claves primarias identifican registros únicos y las foráneas relacionan tablas entre sí. Permiten mantener integridad, evitar duplicados y matrículas inválidas.
2. ¿Cuándo usar procedimientos almacenados o funciones?
Los procedimientos almacenados se usan para procesos complejos, transacciones y modificaciones. Las funciones son ideales para cálculos reutilizables dentro de consultas SQL.
3. ¿Qué rol cumplen índices y estadísticas en e-commerce?
Los índices aceleran búsquedas de productos y categorías. Las estadísticas ayudan al optimizador a generar mejores planes de ejecución.
4. ¿Qué ventajas ofrecen las vistas en un sistema financiero?
Permiten mostrar solo datos necesarios, proteger información sensible, simplificar consultas y controlar permisos.
5. ¿Qué pasa si el log de transacciones se llena constantemente?
Las operaciones pueden detenerse, aumenta el riesgo de pérdida de datos y afecta recuperación y rendimiento general.
6. ¿Qué impacto tiene habilitar autoshrink en producción?
Genera fragmentación de índices, reduce rendimiento y provoca crecimiento y reducción continua de archivos.
7. ¿Qué ocurriría sin restricciones CHECK en pacientes?
Se podrían registrar edades negativas o inválidas, afectando reportes médicos y consistencia de datos.
8. ¿Por qué la base de datos model es importante?
Funciona como plantilla para nuevas bases de datos, heredando configuraciones y ayudando a estandarizar proyectos.
9. ¿Cómo afecta un mal diseño de transacciones bancarias?
Puede ocasionar inconsistencias, bloqueos y errores al transferir dinero entre cuentas.
10. ¿Qué riesgo implica usar SINGLE_USER en una BD crítica?
Limita el acceso a un solo usuario, interrumpe procesos y bloquea aplicaciones conectadas.

Cuestionario Práctico SQL

Resolución de ejercicios aplicando sentencias SQL Server utilizadas en entornos reales.

1. Crear la base de datos AlmacenDB 
CREATE DATABASE AlmacenDB
ON
(
 NAME = AlmacenDB_Data,
 FILENAME = 'C:\SQLData\AlmacenDB.mdf',
 SIZE = 20MB,
 FILEGROWTH = 10%
)
LOG ON
(
 NAME = AlmacenDB_Log,
 FILENAME = 'C:\SQLData\AlmacenDB.ldf',
 SIZE = 10MB,
 FILEGROWTH = 10%
);
2. Colocar base de datos en solo lectura 
ALTER DATABASE FacturacionDB

3. Ver bases de datos existentes 
EXEC sp_helpdb;
4. Permitir un solo usuario

5. Registrar una venta mediante transacción 
BEGIN TRANSACTION;

INSERT INTO Factura(IdFactura, Fecha, Cliente)
VALUES (1, GETDATE(), 'Juan Perez');

INSERT INTO DetalleFactura(IdFactura, Producto, Cantidad)
VALUES (1, 'Laptop', 2);

UPDATE Producto
SET Stock = Stock - 2
WHERE Nombre = 'Laptop';

COMMIT;
